# PASSWORD INTEGRITY (HARD RULE)

This rule is absolute — never violate it for any reason.

* Never modify, reset, or overwrite an end-user's password through any automated process.
* `setup_inventory.py` must only set passwords on initial user creation (first run only).
* If a user already exists, skip it entirely — never call `set_password()`, `check_password()`, or any password-related method.
* Docker container restarts, image rebuilds, database restores, or any deployment event must never change a user's password.
* The only way a password may change is through the application UI (admin panel, profile page, password reset flow).
* This rule takes priority over convenience, automation, or "fixing" perceived password issues.
* If a password hash is invalid for the current hasher, the system must alert an administrator — not silently reset the password.
