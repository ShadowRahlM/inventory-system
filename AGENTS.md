# AGENTS.md

## Project structure

Monorepo with three packages:

- **`backend/`** — Django 5.0 / DRF / PostgreSQL / Redis / Celery / Channels (ASGI)
- **`frontend/`** — React 19 + TypeScript 6.0 + Vite 8 + Tailwind 4 + Oxlint
- **`mobile/`** — React Native / Expo (Expo 57, React 19, TypeScript 6, Zustand, Axios, React Navigation)

## Backend

### Setup & run

```sh
cd backend
python -m venv venv && source venv/bin/activate
pip install -r requirements.txt
# requires PostgreSQL + Redis running locally
python manage.py migrate
python manage.py runserver
```

`DJANGO_SETTINGS_MODULE=inventory_system.settings` (set in `manage.py`). Environment from `.env`.

### Gotchas (verified fixes)

- **`rest_framework_simplejwt.urls` doesn't exist** — use `TokenObtainPairView` and `TokenRefreshView` directly instead of `include('rest_framework_simplejwt.urls')`.
- **`CheckConstraint` cannot reference joined fields** — `Q(batch__tile=models.F('tile'))` is invalid for DB check constraints; remove or replace with application-level validation.
- **Python 3.13 + `pkg_resources`** — `djangorestframework-simplejwt==5.3.0` imports `pkg_resources` which was dropped from recent setuptools. Pin `setuptools<75` if `ModuleNotFoundError: No module named 'pkg_resources'` occurs.

### Architecture

Service-layer: **views → services → models**. Views must be thin, business logic lives in `services.py`, models are data-only.

- `inventory/` is the only Django app
- UUID primary keys on all models
- Every inventory movement creates both a `Movement` and `AuditLog` record atomically (`@transaction.atomic`)
- Inventory tracks `cartons` and `loose_pieces` separately — never mix batches, never round
- Endpoints at `/api/inventory/{tiles, batches, inventory, movements, audit-logs, operations}/`
- **`InventoryViewSet` is `ReadOnlyModelViewSet`** — all mutations go through `InventoryOperationViewSet` actions
- **`MovementViewSet` is `ReadOnlyModelViewSet`** — movements are system-generated by services, never created via API
- **`AuditLogViewSet` is `ReadOnlyModelViewSet`** — audit logs are system-generated
- **Never name a DRF ViewSet action `dispatch`** — it shadows `ViewSet.dispatch()` and breaks request wrapping/permissions. Use `issue_dispatch` instead.
- **Views catch only `ValidationError` and `ObjectDoesNotExist`** from services — never bare `Exception` which masks real errors (transaction failures, DB issues)

### Business-rule enforcement map (`.cursorrules.md` → code)

| Rule | Enforcement | File:Line |
|---|---|---|
| Never mix tile batches | `UniqueConstraint(tile, batch, location)` on Inventory; single-batch FK per operation | `models.py:72-75` |
| Never round quantities | All `IntegerField`, no Decimal/float; integer arithmetic in carton-breakdown | `models.py:60-61` |
| Preserve loose pieces | Dispatch/transfer break cartons → loose pieces (never loose → cartons); loose field tracks remainders | `services.py:126-139, 263-276` |
| Prevent negative stock | `MinValueValidator(0)` on cartons/loose; service pre-checks before deducting; `select_for_update()` prevents race | `models.py:60-61`, `services.py:112,114-121,251,253-258` |
| Track every movement | Every mutation service creates `Movement` record atomically | `services.py:62-75,142-155,202-214,295-307` |
| Every change auditable | Every mutation service creates `AuditLog` record atomically | `services.py:77-89,157-169,216-228,309-325` |
| Inactive batch rejected | All 4 mutation services raise `ValidationError` if batch not active | `services.py:43-44,119-120,263-264` |
| Operations atomic | `@transaction.atomic` on all 4 mutation service methods | `services.py:15,104,189,244` |
| Sales ≤ available stock | dispatch/transfer check total requested ≤ total available pieces before deduction | `services.py:127-131, 266-270` |
| No silent batch metadata drift | `receive_inventory` validates production_date/supplier match when batch already exists | `services.py:42-52` |
| Cartons/loose separate | Two distinct model fields (`cartons`, `loose_pieces`), serialized separately | `models.py:60-61`, `serializers.py:22-31` |
| Views thin, logic in services | Views pass serializer data → service call → response; no business logic in views | `views.py:81-183` |
| No logic in serializers | All operation serializers are pure field definitions (no `validate()` methods) | `serializers.py:58-93` |
| Models data-only | No methods with business logic; only `total_pieces` (computed property) | `models.py:79-84` |

## Frontend

### Commands

```sh
npm run dev         # Vite dev server (port 5173, proxies to backend at 8000)
npm run build       # tsc -b && vite build (typecheck then bundle)
npm run lint        # oxlint (fast Rust-based linter)
npm run preview     # Vite preview of built app
```

**Vite dev expects backend on `http://localhost:8000`** (configured via `VITE_API_URL` env or default in `src/lib/api.ts`).

### Key conventions

- **State**: server state → React Query, app state → Zustand, form state → React Hook Form + Zod
- **Validation**: Zod schemas via `@hookform/resolvers`
- **Strict TypeScript**: `noUnusedLocals`, `noUnusedParameters`, `erasableSyntaxOnly`, `verbatimModuleSyntax`
- **Styling**: Tailwind CSS (v4), no CSS modules or styled-components

### Testing

```sh
npm run test         # vitest run (79 tests across 10 files)
npm run test:watch   # vitest watch mode
```

Uses Vitest + React Testing Library + jsdom. Test files in `src/components/__tests__/`.

## Mobile

### Setup & run

```sh
cd mobile
npm install
npx expo start          # starts Metro bundler, scan QR with Expo Go
npm test                # Vitest (3 tests)
```

### Architecture

- **Navigation**: Conditional stack (Login vs Main tabs) based on `useAuthStore.user`
- **Auth**: JWT stored in AsyncStorage; Axios interceptor attaches `Bearer` token; 401 clears token
- **State**: Zustand `useAuthStore` for auth; all other data fetched fresh (no React Query yet)
- **API layer**: `src/api/client.ts` (Axios instance), `auth.ts`, `tiles.ts`, `inventory.ts`

### Screens

| Screen | Route | Features |
|---|---|---|
| Login | Stack | Username/password, error display, loading indicator |
| Dashboard | Tab | Summary cards (tiles/stock/low-stock), low-stock alert box, tile search |
| Tiles | Tab | Searchable tile list with SKU/name/brand |
| Stock | Tab | Stock levels with low-stock highlighting (red rows below 50 pieces) |
| Movements | Tab | Colored type badges (green/red/amber/indigo), date, reference |

### Key conventions

- All API error handling is silent (no user-facing alerts on load failure)
- Stock items with `total_pieces <= 50` get red background and red text
- Movement type colors: receive=green, dispatch=red, adjust=amber, transfer=indigo
- Tab bar uses emoji icons (no icon library dependency)
- `EXPO_PUBLIC_API_URL` env var for API base URL (defaults to `http://10.0.2.2:8000` for Android emulator)
- TypeScript strict mode enabled

## Production Deployment (Docker)

### Architecture

```
nginx:80 → frontend (React SPA, built static files)
         → /api/* → backend (Django ASGI on :8000) → PostgreSQL + Redis
         → /media/* → backend (static/media files)
         → /ws/* → backend (WebSocket via Channels)
```

### Stack

| Service | Image | Role |
|---|---|---|
| `db` | postgres:16-alpine | Persistent storage |
| `redis` | redis:7-alpine | Cache, Celery broker, Channels layer |
| `backend` | python:3.13-slim | Django ASGI via gunicorn+uvicorn, 4 workers |
| `frontend` | nginx:alpine | Serves built React app, proxies API/media/WS to backend |

### Files

| File | Purpose |
|---|---|
| `backend/Dockerfile` | Python multi-stage build, collects static |
| `frontend/Dockerfile` | Node build → nginx serve |
| `frontend/nginx.conf` | Reverse proxy routes |
| `docker-compose.yml` | Full stack orchestration |
| `.env.production` | Environment variable template |

### Run

```sh
# Build and start
docker compose up -d --build

# Or with custom env
docker compose --env-file .env.production up -d --build

# Stop
docker compose down

# View logs
docker compose logs -f
```

### Configuration

Copy `.env.production` to `.env`, edit values, then:

```sh
# Set a strong secret key
python -c "import secrets; print(secrets.token_urlsafe(50))"

# Update ALLOWED_HOSTS and CORS_ALLOWED_ORIGINS for your domain
```

## Generic rules (from `.cursorrules.md`)

- Never round inventory quantities or mix tile batches
- Prevent negative stock, track every movement, ensure auditability
- Correctness > speed, inventory accuracy > convenience
- Carton-breakdown formula: `loose_pieces += carton_pieces` (add ALL carton pieces) then `loose_pieces -= needed` — never `loose_pieces += (carton_pieces - shortage)` (sign error produces negative stock)
